📋 Expected Column Format
The agent auto-detects these columns (case-insensitive). Only Model ID is required — all others are optional.
* Model ID — hardware part/model code (e.g. FG-60F, WS-C2960X)
Asset — asset tag / alternate model code (fallback)
Vendor / Manufacturer — vendor hint for lookup
Serial number — device serial
Name / Hostname — device display name
Management IP Address — MGMT IP
Company / Site Ref / City — shown as location
Owner / H W Maintained by — owner field
EOL Status — existing EOL value (displayed, not used for lookup)
EOS Date — existing EOS date (displayed alongside our result)
IOS Version — firmware / IOS version string
Hardware Status / Operational status — device status
🔗 ServiceNow CMDB Connection
Connect to your ServiceNow instance to pull live device inventory.
Credentials are used only for this session and never stored.
📖 How ServiceNow Integration Works
1
File: core/servicenow.py — line 30–45 contains the connector class
Enter your instance URL, username, password above and click "Test & Connect"
2
CMDB Table: The agent queries cmdb_ci_netgear for network devices
Fields pulled: name, model_id, model_number, serial_number, ip_address, location, manufacturer
3
Part Code field: Agent uses u_part_code → model_number → model_id (priority order)
Add a custom field u_part_code to your CMDB table for best results
4
Write-back: After analysis, results are written to u_eol_status and u_replacement_sku on each CI
Requires these custom fields to exist on your CMDB table
